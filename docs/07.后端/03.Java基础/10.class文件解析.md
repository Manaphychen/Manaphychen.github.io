我是一个 .java 文件，名叫 FlashObject.java，叫我**小渣**就行。
```java
public class FlashObject {

    private String name;
    private int age;
    
    public String getName() {
        return name;
    }

    public int add(int a, int b) {
        return a + b;
    }

}
```
我马上就要被 JVM 虚拟机老大加载并运行了，此时**老虚**走了过来。<br />老虚：小渣呀，我马上就要把你载了，你先瘦身一下，别占太大地方。<br />小渣：好的，没问题，等我十秒钟。<br />`public class FlashObject{private String name;private int age;public int add(int a,int b){return a+b;}`<br />小渣：老虚，我瘦身好了，你看看。<br />老虚：...，你是不是有病。<br />小渣：怎么了，我把没用的空格和回车啥的都去掉了，瘦身了好多呢！<br />老虚：行吧，看你这智商，我就给你解释解释。你现在仍然是个文本文件，让你瘦身是让你定一个紧凑的数据结构来表示你这个 Java 文件里的信息，然后告诉我这个数据结构中每个字节都代表什么。<br />小渣：哦哦，这样啊。<br />老虚：对啊，这样一是方便我去加载，二是我这个虚拟机可不只是为你 Java 语言服务的，还有很多语言最终都可以转换为我虚拟机识别的，你得设计一个通用的格式。<br />小渣：嗯嗯，这回我明白啦！
# 1. **类信息**
我的**类名**叫 FlashObject。<br />先找个地方把它存起来，放开头吧。<br />![](https://cdn.nlark.com/yuque/0/2022/png/2345482/1658113953058-55329084-ef78-42fa-a232-65a7526ab9da.png#averageHue=%23fbfbfb&clientId=ue201c86c-a9c0-4&from=paste&id=ua3bc00e4&originHeight=225&originWidth=751&originalType=url&ratio=1&rotation=0&showTitle=false&status=done&style=stroke&taskId=ub48428c2-24d4-4aee-b924-25b1348b14c&title=)<br />这里的一个小方格是 1 个字节，也就是 8 位。一个英文字母用 ASCII 码表示为 1 个字节，所以占一个方格，之后不再解释。<br />严谨的我又想到，这个类应该还有其**父类**。<br />虽然这个 .java 文件中没写，但也有其默认父类，Object。<br />当然，我们得记录下全类名<br />java/lang/Object<br />记在哪里呢？就紧跟在类名后面吧。<br />![](https://cdn.nlark.com/yuque/0/2022/png/2345482/1658113953046-80c64126-bd73-40fe-bd49-63d557965549.png#averageHue=%23fbfbfa&clientId=ue201c86c-a9c0-4&from=paste&id=u035a4c02&originHeight=229&originWidth=748&originalType=url&ratio=1&rotation=0&showTitle=false&status=done&style=stroke&taskId=u9b92d10b-3474-4b43-bcf4-a2305acec93&title=)<br />诶不对，我这个类名呀，父类名呀，都是变长的，这样紧挨着放，谁知道分界点在哪。<br />不行不行，得分别在前面加个**长度**，就用两字节表示吧。<br />![](https://cdn.nlark.com/yuque/0/2022/png/2345482/1658113953051-96e2f2ad-f5e3-4b37-a0e2-bd590b7aea1e.png#averageHue=%23fbfbfb&clientId=ue201c86c-a9c0-4&from=paste&id=ua3d010b2&originHeight=233&originWidth=749&originalType=url&ratio=1&rotation=0&showTitle=false&status=done&style=stroke&taskId=ueeed662c-1f93-421a-b25f-54a2970ff2b&title=)<br />除了父类之外，还有**接口名**呢！虽然我们这个类没写，但也得定义出来。<br />这个接口，和类名以及父类名稍有不同，因为可能有多个。<br />但这不是事儿，先占用两个字节，表示接口的数量即可，之后一个一个的接口名仍然像上面那样紧挨着排布。<br />![](https://cdn.nlark.com/yuque/0/2022/png/2345482/1658113953458-f7684258-3655-4058-8bf8-972a905da329.png#averageHue=%23cbdada&clientId=ue201c86c-a9c0-4&from=paste&id=u21fc0f25&originHeight=102&originWidth=585&originalType=url&ratio=1&rotation=0&showTitle=false&status=done&style=stroke&taskId=ua8d44aa5-d1cd-4d97-9ab3-b5a1a17fed7&title=)<br />嗯，完美。
# 2. **常量池**
慢慢地，我发现需要字符串名字的地方越来越多。<br />除了刚刚的类名、父类名、接口名，还有属性名、方法名、属性的类名、方法的入参类型名、返回值类型名，等等等等。<br />一方面，要是每个都这么展开写下去，那文件格式会很乱，很多结构都是变长的。<br />另一方面，很多字符串都是重复的，比如属性 name 的类名 String，与方法 getName 的返回值类名 String，重复写两遍，就浪费了空间。<br />因此，我决定，之前的方案作废，设计一个新的结构来统一存储这些字符串，我给他起名为**常量池**。<br />![](https://cdn.nlark.com/yuque/0/2022/png/2345482/1658113953201-7bd833ce-e063-46a2-8a8f-4d08bc015abd.png#averageHue=%23cedfe0&clientId=ue201c86c-a9c0-4&from=paste&id=ue72ef094&originHeight=93&originWidth=593&originalType=url&ratio=1&rotation=0&showTitle=false&status=done&style=stroke&taskId=ue33ca178-6147-4b1e-8522-b0517b7d237&title=)<br />每个字符串都有一个**索引**与之对应，这个是可以计算出来的，不需要额外的字段。<br />![](https://cdn.nlark.com/yuque/0/2022/png/2345482/1658113953496-59342475-2a24-41bb-bb1a-4cabb9c76326.png#averageHue=%23efc091&clientId=ue201c86c-a9c0-4&from=paste&id=u689aeebf&originHeight=151&originWidth=592&originalType=url&ratio=1&rotation=0&showTitle=false&status=done&style=stroke&taskId=ua8e17aa4-f3a6-4ec1-be5e-d5c1b16eca5&title=)<br />这样，刚刚的类、父类、接口，就都可以指向这个索引了，也因此可以将长度固定下来。<br />![](https://cdn.nlark.com/yuque/0/2022/png/2345482/1658113953515-a66d98e0-407c-4c38-be05-c3bafc49b888.png#averageHue=%23f0d2ad&clientId=ue201c86c-a9c0-4&from=paste&id=u1296b853&originHeight=268&originWidth=788&originalType=url&ratio=1&rotation=0&showTitle=false&status=done&style=stroke&taskId=u5e64cd89-15da-4bbf-9eb8-7c526da2151&title=)<br />当然，现在这个常量池，仅仅存放了字符串。<br />不难想到，还可能有整型、浮点型的值作为常量，甚至还有可能是个引用类型，然后这个引用类型再次指向常量池中的一个索引，有点像指针的指针。<br />那这么多类型，必然就还需要一个记录**类型信息**的地方，看来我们得将之前的设计改改。<br />![](https://cdn.nlark.com/yuque/0/2022/png/2345482/1658113954363-ee615ff7-f9b3-4827-9218-00be2bf91013.png#averageHue=%23faf4ed&clientId=ue201c86c-a9c0-4&from=paste&id=u97a73f3e&originHeight=393&originWidth=622&originalType=url&ratio=1&rotation=0&showTitle=false&status=done&style=stroke&taskId=u73387c95-8cbc-4fb2-8250-83687c50868&title=)<br />这样，我们的常量池，就不单单可以存储简单的字符串常量了，而是可以根据不同类型，存储与其相对应的数据结构的值。<br />当然，我们常量池的整体结构还是不变的，只不过里面是类型丰富的结构。<br />![](https://cdn.nlark.com/yuque/0/2022/png/2345482/1658113953553-91b8c8d0-bd9f-4bfb-a7ef-743426005482.png#averageHue=%23e7e5e3&clientId=ue201c86c-a9c0-4&from=paste&id=u457c63fa&originHeight=123&originWidth=511&originalType=url&ratio=1&rotation=0&showTitle=false&status=done&style=stroke&taskId=ucf688435-5a43-44ab-b443-9139b40402b&title=)<br />同样，我们的整个设计，也没有因为常量池的小改动，受到影响。<br />![](https://cdn.nlark.com/yuque/0/2022/png/2345482/1658113953822-034ad8c3-911f-40ea-a72d-ece0bee30a51.png#averageHue=%23f0d2ad&clientId=ue201c86c-a9c0-4&from=paste&id=uf05080d9&originHeight=268&originWidth=788&originalType=url&ratio=1&rotation=0&showTitle=false&status=done&style=stroke&taskId=u791195b8-49d1-47b1-b10e-23fdcf24e01&title=)<br />OK，总结一下我们目前的整体方案。<br />开头存**常量池**，之后需要的常量就全往这里放，用一个索引指向它即可。<br />紧接着存放**类本身**的相关信息，我们存放了**当前类、父类以及接口**的信息。<br />看来老虚要求的瘦身工作，已经初具规模啦。
# 3. **变量** 
现在类本身的信息，已经找到合适的位置存放起来了，接下来我们存变量。<br />变量也可能有多个，所以结构依然仿照我们之前的思路，开头存数量，后面紧跟着各个存放变量的数据结构。<br />![](https://cdn.nlark.com/yuque/0/2022/png/2345482/1658113953800-6da01fbc-e3bf-42c5-96eb-049a065b195a.png#averageHue=%23d5d6d2&clientId=ue201c86c-a9c0-4&from=paste&id=uca7b34b9&originHeight=89&originWidth=511&originalType=url&ratio=1&rotation=0&showTitle=false&status=done&style=stroke&taskId=u3250f711-7381-4649-b6e8-105606288b1&title=)<br />至于变量用什么数据结构来存，是不是定长的，那就是我们接下来要设计的了。<br />我们把其中一个变量拿出来，看看它有什么？
:::info
private String name;
:::
非常清晰，private 这部分是变量的标记，String 是变量类型，name 是变量名字。
## **先看标记部分**
除了 private，还有 public、protected、static、final、volatile、transient 等，有的可以放在一起，比如
:::info
public static final String name；
:::
有的不能放在一起，比如
:::info
public private String name; //错误
:::
我们用**位图**的方式，每一个标记用一个位来表示（比如 public 在第一个位，private 在第二个位，static 在第四个位，final 在第五个位...），这样不论如何排列组合，最终的值都是不一样的。<br />![](https://cdn.nlark.com/yuque/0/2022/gif/2345482/1658113954090-726c3514-d8d4-4dd1-95f6-cd3b0a7ec779.gif#averageHue=%23f5f5f5&clientId=ue201c86c-a9c0-4&from=paste&id=u85e79781&originHeight=241&originWidth=551&originalType=url&ratio=1&rotation=0&showTitle=false&status=done&style=stroke&taskId=ubf41b384-14ba-4cc4-89a4-ad9b1c11a69&title=)<br />我们把这些标记所对应的值，都设计并记录下来。<br />复合型的标记，就可以表现为将其相加，比如 public static，就是 0x0001 + 0x0008 = 0x0009。

| **标记** | **值** |
| --- | --- |
| public | 0x0001 |
| private | 0x0002 |
| protected | 0x0004 |
| static | 0x0008 |
| final | 0x0010 |
| volatile | 0x0040 |
| transient | 0x0080 |

而这样的赋值方式，不同排列组合后的和没有重复的，且也能根据值很方便地反推出标记。<br />不错不错，就这样了。<br />哦对了，类信息本身也有 public 呀 private 这些标记属性，刚刚记录类信息的时候忘了，先加上它，免得一会忘了！<br />![](https://cdn.nlark.com/yuque/0/2022/png/2345482/1658113954119-3e8e43ed-2792-4b79-9896-ecc93d5e1685.png#averageHue=%23fafafa&clientId=ue201c86c-a9c0-4&from=paste&id=u88e1bda5&originHeight=265&originWidth=786&originalType=url&ratio=1&rotation=0&showTitle=false&status=done&style=stroke&taskId=u5ce65bf7-f76d-422a-ba37-6e9d779d526&title=)
## **再看类型部分**
当前类型为 String，属于一个引用数据类型中的**类类型**。
:::info
private String name;
:::
除此之外，还有**八个基本数据类型**，和引用类型中的**数组类型**。<br />为了占用更少的空间，我们将其用最少的符号来表示。<br />这里的基本数据类型，和数组类型，都只占用一个 char 来表示，就只占了 1 个字节。

| **符号表示** | **类型** |
| --- | --- |
| B | byte |
| C | char |
| D | double |
| F | float |
| I | int |
| J | long |
| S | short |
| Z | boolean |
| L_ClassName_ ; | 类 |
| [ | 数组 |

如果是类，则占用了 L 和 ; 两个字节，再加上全类名所占的字节数。<br />比如这里的 String 类型，用符号表示，就是<br />Ljava/lang/String;<br />但注意，这里的符号，也都可以存放在常量池中，而我们的变量结构中的类型描述符部分，只需要一个常量池索引即可。<br />![](https://cdn.nlark.com/yuque/0/2022/png/2345482/1658113954143-5a1ffe2a-5f4f-4338-a3c8-c2aaed11bd0f.png#averageHue=%23f0eae4&clientId=ue201c86c-a9c0-4&from=paste&id=u1386bf75&originHeight=403&originWidth=458&originalType=url&ratio=1&rotation=0&showTitle=false&status=done&style=stroke&taskId=u435e40a3-bfed-40ee-99a3-5988570f1ab&title=)<br />ok，第二部分也搞定了。

名字部分没什么好说的，相信你直接能猜到了，直接上图。<br />![](https://cdn.nlark.com/yuque/0/2022/png/2345482/1658113954309-7d968f80-c1b1-4da0-a7e1-dcc227123cfe.png#averageHue=%23eee7e2&clientId=ue201c86c-a9c0-4&from=paste&id=u5df521e8&originHeight=408&originWidth=466&originalType=url&ratio=1&rotation=0&showTitle=false&status=done&style=stroke&taskId=uc9a0ee10-2253-45b7-a3d1-6e34f088f95&title=)<br />OK，**两字节的标记、两字节的类型描述符、两字节的变量名称**，这个就是我们一个变量的数据结构。<br />![](https://cdn.nlark.com/yuque/0/2022/png/2345482/1658113954533-47099e8a-d2fe-4703-98cd-f872897f6ea5.png#averageHue=%23eabc8d&clientId=ue201c86c-a9c0-4&from=paste&id=u20515a9f&originHeight=268&originWidth=541&originalType=url&ratio=1&rotation=0&showTitle=false&status=done&style=stroke&taskId=u966d6d47-052e-440a-b389-f570811be62&title=)<br />把它放到我们最终的总视图里。<br />![](https://cdn.nlark.com/yuque/0/2022/png/2345482/1658113954598-d0dc198b-dfc2-46a0-84fe-bebc65526024.png#averageHue=%23f0dbc3&clientId=ue201c86c-a9c0-4&from=paste&id=ub04ae564&originHeight=301&originWidth=787&originalType=url&ratio=1&rotation=0&showTitle=false&status=done&style=stroke&taskId=u52be05df-31f0-4858-aaa1-1ca857b7059&title=)<br />搞定！
# 4. **方法**

```java
@RequestMapping()
public String function(String a) throws Exception {
    return a;
}
```
那就会有代码部分、异常、注解等需要录入的信息。<br />但似乎除了代码部分之外，其他部分都不是每个方法都有的，如果都定义出来，岂不是浪费空间，那怎么办呢？<br />我们效仿常量池的做法，把这些部分都叫“**方法的属性**”，一个方法可能有多个属性，设计结构如下。<br />![](https://cdn.nlark.com/yuque/0/2022/png/2345482/1658113954701-0ed78fab-d91b-44fa-acf2-ac8d2c216a72.png#averageHue=%23f3c393&clientId=ue201c86c-a9c0-4&from=paste&id=ued2c411b&originHeight=540&originWidth=520&originalType=url&ratio=1&rotation=0&showTitle=false&status=done&style=stroke&taskId=uacc272e7-9d91-43ec-988d-e593053e1b5&title=)<br />这样，方法具有哪些属性，**按需添加**进来就好，如果不需要这个属性，也不用浪费空间，完美！<br />回过头看我们的这个方法。
```java
public int add(int a, int b) {
    return a + b;

```

# **6. 后记**
根据 Java 虚拟机规范，Java Virtual Machine Specification Java SE 8 Edition，一个 class 文件的标准结构，是这样的。
```java
ClassFile {
    u4             magic;
    u2             minor_version;
    u2             major_version;
    u2             constant_pool_count;
    cp_info        constant_pool[constant_pool_count-1];
    u2             access_flags;
    u2             this_class;
    u2             super_class;
    u2             interfaces_count;
    u2             interfaces[interfaces_count];
    u2             fields_count;
    field_info     fields[fields_count];
    u2             methods_count;
    method_info    methods[methods_count];
    u2             attributes_count;
    attribute_info attributes[attributes_count];
}
```
我们的设计与它几乎相同。<br />只有后两项，我们没有涉及到，本身也不是重点。<br />常量池中的类型，有以下几种。
